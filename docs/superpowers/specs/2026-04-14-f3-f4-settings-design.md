# F-3/F-4 设置页整合设计（模型配置 + 领域管理）

日期：2026-04-14  
范围：PunkRecords 前端设置信息架构与交互设计（不含实现代码）

## 1. 背景与目标

当前 UI 中：

- 左侧导航存在“模型配置”独立入口；
- “设置”入口实际承载的是领域管理；
- F-3（模型参数）与 F-4（Vault 路径）尚未形成统一设置体验。

本设计目标：

1. 将用户可配置项统一收敛到“设置”页；
2. 实现 F-3 的可编辑模型配置（手动保存）；
3. 实现 F-4 的路径配置，并与领域管理合并；
4. 明确“简洁、美观”的视觉与布局约束，避免功能堆叠导致可用性下降。

## 2. 信息架构（IA）

### 2.1 左侧导航

左侧导航调整为仅两项：

- 主页
- 设置

说明：

- 移除左侧“模型配置”独立入口；
- 用户菜单中也不再单独暴露“模型配置”，统一通过“设置”进入。

### 2.2 设置页内 Tab

设置页采用 Tab 结构，共两个分区：

1. 模型配置
2. 领域管理

其中“领域管理”Tab 合并承载：

- 领域新增/编辑/归档；
- 全局路径配置；
- 每领域路径覆盖配置。

## 3. F-3 模型配置设计

### 3.1 字段

- `llm_provider`：下拉可编辑，可配置并保存；
- `llm_model`：文本输入；
- `llm_base_url`：URL 输入；
- `llm_api_key`：密码输入（掩码）。

### 3.2 保存策略

- 采用手动保存；
- Tab 内提供“保存模型配置”按钮；
- 仅提交有变更字段。

### 3.3 安全与展示

- 首次加载不回显 API Key 明文；
- 若用户未修改 API Key，提交时不覆盖；
- 仅当用户输入新值时更新 API Key。

### 3.4 校验与反馈

- `provider` 必填；
- `model` 必填；
- `base_url` 可空，非空时必须是合法 URL；
- 保存中禁用按钮并显示加载态；
- 成功提示“模型配置已保存”；
- 失败显示可理解错误信息（优先映射后端 code/message）。

## 4. F-4 领域管理 + 路径配置设计

### 4.1 结构

“领域管理”Tab 包含两块卡片：

1. 全局路径配置卡片
2. 领域列表与覆盖配置卡片

### 4.2 路径规则

- 全局字段：`materials_vault_path`；
- 每领域可配置覆盖路径；
- 领域覆盖路径为空时，继承全局路径；
- 界面文案明确：留空 = 使用全局路径，填写 = 覆盖该领域路径。

### 4.3 保存策略

- “领域管理”Tab 使用统一保存按钮；
- 一次提交领域元数据与路径覆盖变更；
- 减少多次提交导致的状态分裂。

### 4.4 校验与异常

- 路径输入自动 trim；
- 前端执行基础格式校验；
- 后端负责存在性/可写性等最终校验；
- 错误尽量行内展示，减少阻断弹窗。

## 5. 视觉与交互约束（简洁、美观）

### 5.1 布局层级

- 顶部：页标题 + Tab + 主操作；
- 内容区：卡片分组呈现（全局路径卡、领域列表卡）。

### 5.2 视觉基调

- 延续白底、浅灰边框、轻阴影；
- 主操作色使用既有橙色；
- 避免重装饰、避免过密信息表格。

### 5.3 组件规范

- 领域列表采用“行卡片”布局；
- 输入框/按钮统一高度与圆角；
- 间距采用 8/12/16 节奏；
- 窄屏下路径输入优先占满宽度，操作按钮可换行。

### 5.4 反馈策略

- 行级错误就近提示；
- 全局成功采用轻量 toast；
- 保存过程有明确 loading 状态与禁用态。

## 6. 数据流建议（接口层）

建议接口演进（命名可按现有后端约定微调）：

- `GET /api/v1/settings/llm`：返回模型配置；
- `PATCH /api/v1/settings/llm`：保存模型配置；
- `GET /api/v1/settings/domains`：返回全局路径与领域覆盖路径；
- `PATCH /api/v1/settings/domains`：保存领域管理与路径配置（支持批量）。

约束：

- 未登录不可进入设置能力；
- 与首登路径确认（F-1）复用相同路径校验语义；
- 错误码需稳定，前端才能做友好映射。

## 7. 范围与非目标

本次设计范围：

- IA 调整；
- 设置页 Tab 结构；
- F-3/F-4 交互、校验、反馈；
- 视觉简洁性约束。

本次非目标：

- 后端热更新策略细化；
- 任务系统（异步作业）；
- 图谱入口与会话历史改造。

## 8. 验收标准（设计层）

1. 导航中不再出现独立“模型配置”入口；
2. 设置页内可见“模型配置 / 领域管理”两个 Tab；
3. 模型配置字段可编辑并手动保存；
4. 领域管理页同时可编辑全局路径与每领域覆盖路径；
5. 领域管理页视觉风格符合“简洁、美观”约束。
